# USER\_STORY.md

* User Story 1：建立小組與比對空堂

作為一位剛被分好組的大學生，我想要在平台上與新組員加為好友並拉入同一堂課的小組中，因為這樣系統就能自動比對出我們的共同空堂，我也能在自己的課表介面中設定打工、社團等「不可用時間」，因為我需要確保系統計算出的共同空堂，確實是我能安心參與討論的空閒時段，不需再浪費時間手動核對每個人的作息。

* User Story 2：時段投票決策

作為一位小組長，我想要在系統算出多個共同空堂後，直接在介面上勾選幾個時段發起「投票」，並由系統自動通知組員投票，並告知結果。因為我不想再把結果截圖丟到 LINE 群組裡，慢慢統計大家到底想選哪一個。
