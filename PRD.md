# PRD.md

\-專案概述 (Project Overview)
TeamSync 是一個前端靜態網站的展示 Demo。主要目的是解決大學生在修習跨系所課程時，為了尋找小組討論時間所面臨的「溝通門檻」與「比對地獄」。本專案將透過直覺的介面與核心演算法（陣列交集或位元運算），一鍵精準找出小組成員皆有空的時段。此專案專為課堂 Demo 設計，不包含真實後端、資料庫與登入系統。

\-目標受眾與使用情境 (Target Audience \& Scenarios)
目標受眾：需要頻繁進行小組報告、跨系所修課的大學生。核心痛點：每個人課表不同，且有額外的打工、社團等私人行程，傳統上透過文字或截圖來回比對空堂非常耗時且容易出錯。

\-用戶故事 (User Stories)
Story 1: 個人時間設定與小組媒合作為一位剛被分好組的大學生，我想要在平台上與新組員加為好友並拉入同一堂課的小組中，因為這樣系統就能自動比對出我們的共同空堂。我也能在自己的課表介面中設定打工、社團等「不可用時間」，因為我需要確保系統計算出的共同空堂，確實是我能安心參與討論的空閒時段，不需再浪費時間手動核對每個人的作息。
Story 2: 特定課程的小組視角作為一位修習多門課程的學生，我想要點進特定課程的介面時，能清楚看見該堂課的小組成員名單，因為我需要快速確認不同課程的合作對象，並隨時查詢該組的共同可討論時間。

\-核心功能規格 (Core Features / MVP)
4.1 個人課表與不可用時間管理功能描述：提供視覺化的週曆介面，展示使用者的固定課程。互動設計：使用者可以點擊特定的空白時段，將其標記為「不可用時間」（如打工、社團）。預期結果：更新後的個人可用時間資料將即時反映在後續的共時計算中。
4.2 課程與小組成員管理介面功能描述：提供一個清單或區塊，列出使用者目前修習且需要分組的「特定課程」。互動設計：使用者點擊某一門課程後，畫面需切換或展開，顯示該課程的「小組成員清單」（包含好友/組員頭像或名稱）。
4.3 共時排程引擎 (核心亮點)功能描述：系統自動匯集所選小組成員的課表與不可用時間，進行交集運算。互動設計：使用者點擊「尋找共同空堂」按鈕後，系統需一鍵顯示計算結果。視覺呈現：運用視覺化疊加動畫或直覺的區塊顏色變化（例如：共同空堂顯示為高亮綠色），精準 Highlight 出大家都能配合的時段。
4.4 演算法與資料結構說明展示功能描述：為了滿足課堂 Demo 需求，介面上需有一個區塊或按鈕（例如「運算邏輯說明」），清楚解釋本系統是如何使用 Array（陣列交集）或 Bitwise（位元運算）來達成排程邏輯。

\-演算法與資料設計建議 (Algorithm \& Data Structure)
為了滿足「清楚說明資料結構與運算規則」的驗收標準，建議採用以下兩種方式之一來設計 data/schedules.js 的排程邏輯：方案 A (位元運算 Bitwise)：將一天劃分為多個時段（例如 14 堂課）。每個人的單日課表用一個二進位數字表示（例如 11100011110000，1 代表有空，0 代表沒空）。比對時只需將所有組員的數字進行 Bitwise AND (\&) 運算，即可快速得出共同空堂。方案 B (陣列交集 Array Intersection)：將每個人的可用時段轉化為字串或 ID 陣列（例如 \["Mon-1", "Mon-2", "Tue-3"]），透過 JavaScript 的 filter 與 includes 方法找出存在於所有成員陣列中的共同元素。
