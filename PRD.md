# Product Requirement Document (PRD) - TeamSync 跨堂分組共時排程引擎

## 1. 專案背景與定位 (Project Background)
在大學課程中，學生經常面臨跨系所選課、打工與社團活動等時間衝突，導致小組難以找出共同討論時間。本專案「TeamSync」旨在打造一個輕量、直觀且運算高效的「共時排程交集引擎」，解決多人口頭或手動對話比對時間的溝通門檻。
本專案為**前端展示 Demo 系統**，重點展示核心資料結構、位元運算演算法，以及直觀的空堂高亮 UI，不串接真實後端資料庫。

---

## 2. 核心功能需求 (Product Features)
1. **個人課表與忙碌狀態檢視**：
   * 學生能檢視自己的週一至週五課表。
   * 提供開關切換學生的「自訂忙碌時間」（如社團活動、兼職打工）。當開關切換時，課表需動態響應。
2. **好友關係展示**：
   * 顯示目前操作學生的好友名單，展現關聯式設計中的社群網路雛形。
3. **課堂小組成員檢視**：
   * 使用者選取特定課程的小組後，系統需載入並顯示該小組的成員名單。
4. **一鍵比對共時空堂 (Core)**：
   * 使用者選取小組後，點擊「一鍵比對」，系統自動匯集所有組員的課表與自訂忙碌時間。
   * 計算出所有人皆為空閒的「共同時段」，並以高亮視覺動畫標示於課表 Grid 中。
5. **架構運作即時解析 (Architect Visualizer)**：
   * 提供即時的位元運算運作日誌，顯示各組員的 Busy Mask（14-bit）與計算過程，將演算法視覺化。

---

## 3. 非功能性需求 (Non-Functional Requirements)
1. **零安裝成本**：作品必須為靜態網頁，雙擊 `index.html` 即可在瀏覽器中直接開啟，不需要任何 Web 伺服器、資料庫或 npm 套件。
2. **高效能比對**：採用位元運算 (Bitwise Operations) 處理交集，確保時間複雜度為 $O(N)$，空間複雜度為 $O(1)$。
3. **視覺反饋**：使用深色科技感介面，共同空堂高亮需有漸層與微光起動畫，符合報告展示需求。
