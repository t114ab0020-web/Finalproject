關於此專案

本專案為跨堂分組共時排程引擎前端展示系統。

為解決大學生找時間讨论時的溝通成本，系統使用位元運算 (Bitwise Operations)，將 14 個時段壓縮成一個整數遮罩，以 $O(N)$ 時間複雜度一鍵求得小組交集。

位元運算邏輯解析器 (Architect Visualizer)

將各組員 14 個時段的忙碌狀態，透過位元運算快速比對

🎯 位元運算核心公式

// 1. 合併各組員忙碌遮罩，取得小組忙碌聯集 (OR 運算)
groupBusyMask = member_1_Busy | member_2_Busy | ... | member_n_Busy;

// 2. 取反並以 14-bit 遮罩 (0x3FFF) 限制，求出共同空堂交集 (NOT 運算)
groupFreeMask = (~groupBusyMask) & 0x3FFF;

📊 為什麼使用位元運算資料結構？

評估維度 傳統陣列交集 (Array Intersection) 位元運算遮罩 (Bitwise Mask) 🚀
資料結構 Array of Booleans 或 Objects (佔用較多記憶體) 14-bit Integer (僅佔 1 個整數空間，極度輕量)
資料庫存儲 需要多張關聯表或 JSON 字串，不易進行 SQL 篩選 在 student_schedules 中存為單一 INT 欄位，可用 SQL 位元查詢
比對演算法 巢狀迴圈比對：$O(M \times N)$，需逐格檢查並過濾 位元或與反運算：$O(N)$，利用 CPU 暫存器單一指令即可完成 14 時段運算
擴充彈性 隨時段增加，陣列操作與渲染開銷呈線性增長 在 32 位元/64 位元整數限制內（可支持高達 64 個時段），運算速度維持常數時間
